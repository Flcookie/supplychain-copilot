# Supplier Management Workbench — Demo 改造方案

> 定位：把现有纯聊天 Demo 升级为"供应商管理工作台 + AI Copilot"，让观众第一眼就觉得这是真实业务产品，而不是课程作业。
>
> 约束：**后端不动**（graph/、core/、rag/、tools/ 全部保留），只扩展 `app/` 目录和 Streamlit 页面结构。

---

## 一、现状诊断

### 当前问题

| 维度 | 现状 | 业务产品应有的样子 |
|------|------|------------------|
| 入口 | 打开即是聊天框，没有上下文 | 有 Dashboard，一眼知道"今天有什么需要处理" |
| 信息架构 | 单页全量聊天 | 多页：供应商列表、KPI 看板、审查队列、准入流程 |
| 交互模式 | 用户必须知道怎么问 | 提供可点击的卡片、按钮，AI 是辅助而非入口 |
| 供应商感知 | 只有 SUP012 这类 ID | 有供应商名称、类别、状态徽章、风险颜色 |
| AI 位置 | AI 即产品本身 | AI 是右侧 Copilot 浮层，配合当前页面上下文 |
| 动作反馈 | 只能看文字回答 | 有"发起复审""标记风险""导出报告"等操作按钮 |

### 现有后端能力（已有，直接复用）

- `kpi_query` → 已有 SQLite + 模板 SQL，可驱动 KPI 卡片和图表
- `risk_scenario` → 已有风险规则，可驱动"需要处理"列表
- `qualification_checklist` → 已有清单生成，可驱动准入向导
- `vendor_rating_explanation` → 已有评分解释，可驱动供应商详情页
- `policy_qa` → 已有 RAG，可驱动政策问答入口

**定位**：把现有的"问一个问题得到清单"改成向导式 3 步流程，更像真实采购系统。

  🤖 根据你填写的信息，AI 生成以下准入要求：

  必须文件                        状态
  ✅ ISO 9001 质量管理认证
  ✅ ESG 自评报告（Ratti 模板）
  ⬜ 中国纺织品出口许可证
  ⬜ REACH 合规声明
  ⬜ 供应商行为准则签署

  [← 返回]  [确认清单，进入下一步 →]

─────────────────────────────────────────────
Step 3 / 3  创建供应商档案              ● ● ●

## 五、AI Copilot 浮层设计

### 位置与触发

```
主内容区（90%宽度）         │  Copilot Panel（10% → 展开 35%）
                           │
                           │  [🤖 Copilot  ✕]
                           │  ─────────────────
                           │  上下文: 当前页面 = 供应商详情
                           │  供应商: TechFab Italia (SUP021)
                           │
                           │  快速问题:
                           │  [为什么 C 评级?]
                           │  [复审建议是什么?]
                           │  [查 OTD 趋势]
                           │
                           │  ──────────────────
                           │  [输入你的问题...]
                           │
                           │  答: TechFab 的评级降级...
                           │  来源: Vendor Rating Policy §3.2
```

### 上下文感知

每个页面传入不同的"上下文前缀"，让 Copilot 回答更聚焦：

| 当前页面 | 注入上下文 |
|---------|----------|
| Supplier 详情页 SUP021 | 自动预填 `supplier_context = "SUP021 TechFab Italia"` |
| Review Queue 某条任务 | 预填 `"Focus on risk review for SUP021"` |
| Qualification Step 2 | 预填已填写的类别/国家信息 |
| Policy 页 | 无额外上下文，纯政策问答 |

实现方式：在调用 `run_copilot()` 时拼接上下文前缀到 `question`，后端无需改动。

## 十一、关键设计原则

1. **AI 是助手，不是入口**：用户通过界面导航做任务，AI 是旁边的"智能同事"
2. **数据先于对话**：每个页面都有可见的数据/状态，不需要问问题才能了解现状
3. **动作明确**：每个卡片/行都有明确的下一步按钮（发起复审、Ask AI、发送提醒）
4. **人机协同可见**：`human_approval_required` 警告在 UI 里有专属样式，不只是文字
5. **一页一个用户任务**：每个页面对应一个业务角色的真实工作任务，不是技术功能划分
